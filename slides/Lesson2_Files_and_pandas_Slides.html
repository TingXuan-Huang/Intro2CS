From files to trustworthy computation

Native Python I/O, pandas, PyArrow, H5AD, and NumPy

Reading a file is a translation step. Analysis begins only after we inspect what the translation produced.
0 · Why files?

Memory is temporary; files make data persistent

In memory

orders = [
  {"id": "T1", "amount": 100},
  {"id": "T2", "amount": 150},
]

Fast to use, but the object disappears when the process ends.

On disk

path.write_text(
  "T1,100\nT2,150\n",
  encoding="utf-8"
)

Slower to access, but reusable, shareable, and auditable.

Persistence introduces a boundary: an in-memory object must be encoded into bytes, then reconstructed later.

1 · Paths

Ask three questions

  • What route did I construct?
  • Does that resource exist?

1 · Diagnose the boundary

FileNotFoundError is usually an address problem

If the notebook runs from notebooks/, which file does Path("../data/raw/customerz.csv") describe?
try:
    pd.read_csv(
        Path("../data/raw/customerz.csv")
    )
except FileNotFoundError as error:
    print(error)
Read the final traceback line first. Then print Path.cwd(), print the path, and call path.exists(). The typo is in the filename.
Do not change the data-loading code until you have verified the path itself.
1 · Resource lifetime

A context manager closes the file even when work fails

path = Path("notes.txt")

with path.open(
    mode="r",
    encoding="utf-8"
) as file:
    text = file.read()

# file is closed here

The ownership pattern

with opens the resource, gives it a local name, and guarantees cleanup when the indented block ends.

Use it for files, database connections, locks, and other managed resources.

Safe I/O is not only about content; it is also about who owns the open resource and when it is released.

1 · Text decoding

Encoding is the agreement that connects characters and bytes

Text mode

path.open(
  "r",
  encoding="utf-8"
)

Python decodes bytes into a str.

Binary mode

path.open("rb")

Python returns bytes; the caller must interpret them.

Why specify UTF-8?

Platform defaults differ. An explicit encoding turns an environmental guess into part of the program's contract.

1 · Reading strategies

Choose a reading pattern from file size and processing needs

Whole file

text = file.read()

Simple when the file comfortably fits in memory.

One line

line = file.readline()

Useful when a header or first record has special meaning.

Stream lines

for line in file:
    process(line)

Uses bounded memory and supports incremental work.

“Load everything” is a convenience, not a universal requirement.
1 · Text round trip

Write, reload, and compare what actually persisted

path = Path("first-file.txt")
expected = "alpha\nbeta\n"

path.write_text(expected, encoding="utf-8")
actual = path.read_text(encoding="utf-8")

print(repr(actual))
assert actual == expected
print(path.stat().st_size, "bytes")

What the check catches

  • Wrong output path
  • Missing newline characters
  • Unexpected encoding behavior
  • Partial or empty output
Saving without reloading tests only that the write call finished—not that another program can reconstruct the intended information.

2 · CSV and TSV

Delimited text represents a rectangle—but not its full schema

What the text tells us

  • Rows, fields, and a delimiter
  • Quoting and newline rules

What it does not guarantee

  • Column dtypes or valid ranges
  • Key uniqueness
  • Whether 00101 is an ID or number
CSV preserves characters in a rectangle. Meaningful column contracts must be supplied or inferred.

2 · JSON

JSON preserves hierarchy, not arbitrary Python object identity

import json

with path.open(encoding="utf-8") as file:
    data = json.load(file)

with out.open("w", encoding="utf-8") as file:
    json.dump(data, file, indent=2)

Natural JSON values

Objects, arrays, strings, numbers, booleans, and null.

Round-trip changes

Tuples reload as lists. Sets and datetimes require an explicit conversion. Object keys are strings.

JSON is a good transport for nested records, but it is not a complete serializer for every Python type.
2 · Nested data

Before flattening JSON, decide what one output row means

{
  "portfolio": "P1",

Should one row represent a portfolio, a holding, or a trade?
For trade-level analysis, one row should represent one trade. Portfolio and ticker fields are repeated as context.
Flattening is a modeling decision, not a mechanical conversion.
2 · Excel

An Excel file is a workbook, not merely a table

Useful human features

  • Multiple named sheets
  • Formulas, formatting, and notes
  • Convenient manual review

Parsing complications

  • Decorative title rows and merged cells
  • Different layouts across sheets
  • Values that depend on workbook recalculation
Use Excel for human workflows; inspect workbook structure before assuming a clean rectangular table.
2 · Parquet

Parquet stores typed columns for analytical access

Schema

Logical column names and types are stored with the data.

Columnar layout

Readers can request only the columns needed for a calculation.

Compression

Repeated values within columns can be encoded efficiently.

Important qualification

Parquet preserves supported table types and schema information; it does not preserve arbitrary Python objects or every pandas-specific detail in every engine.

3 · PyArrow Parquet I/O

Column projection avoids decoding data the computation will not use

import pyarrow.parquet as pq

pq.write_table(
    table,
    "sales.parquet",
)

subset = pq.read_table(
    "sales.parquet",
    columns=["customer_id", "revenue"],
)

result = subset.to_pandas()

Why this matters

A columnar file can skip entire unused columns. For wide analytical tables, this reduces I/O and decoding work.

Filtering and partitioning can reduce the read further.

PyArrow is especially useful when storage and data movement—not only DataFrame manipulation—are part of the problem.

3 · Division of labor

pandas is analysis-facing; Arrow is interchange-facing

Stay in pandas when you need

  • Index-aware selection and alignment
  • Cleaning, grouping, joining, reshaping
  • Convenient exploratory output

Use PyArrow when you need

  • Explicit columnar schemas
  • Parquet datasets and selective reads
  • Efficient interchange with other systems
These libraries complement each other; one is not a universal replacement for the other.

4 · H5AD hierarchy

One H5AD file stores several aligned structures

Matrix-aligned

X and layers/* have shape n_obs × n_vars.

Observation-aligned

obs, obsm, and obsp follow observations.

Variable-aligned

var, varm, and varp follow variables.

Unstructured

uns stores nested metadata without an axis-shape contract.

Logical hierarchy shown. An HDF5 table or sparse matrix may itself be encoded as a group rather than one leaf dataset.

4 · Read and write H5AD

Scanpy reads the coordinated object; AnnData writes it

Scanpy workflow

Role distinction

AnnData defines the object and H5AD I/O. Scanpy adds a single-cell analysis workflow around that object.

4 · Large H5AD files

Backed mode keeps the main matrix on disk—but changes the rules

Do not treat backed mode like normal memory

  • Data is loaded from disk on demand.
  • Mutation support is restricted.
  • Write edits to a new file unless you fully understand the backing rules.
Backed mode is a memory strategy, not a faster default for every dataset.

5 · Silent failure

The wrong dtype can answer the wrong question without crashing

df["revenue"] = (
    df["quantity"]
    * df["unit_price"]
)

# unit_price is text

Why this is worse than an exception

The operation finishes. The output column exists. Downstream code continues. A plausible-looking report may be produced.

Cleaning must come before arithmetic.

Before computing, verify that the operation matches both the dtype and the real-world meaning.

5 · Text normalization

Normalize categories before filtering, grouping, or validating

customers["region"] = (
    customers["region"]
    .astype("string")
    .str.strip()
    .str.title()
)

valid = {"East", "West", "North", "South"}

Common string tools

  • str.strip() removes surrounding whitespace
  • str.lower() or str.title() standardizes case
  • str.replace() repairs known patterns
  • isin(valid) tests the approved set
Normalization must be documented; automatic case changes are not correct for every identifier or name.
5 · Range checks

An anomaly is a question—not automatically an error

suspicious = df.loc[

        "quantity",

Decision sequence

  1. Notice the value.
  2. Investigate its source and context.
  3. Decide whether it is valid, missing, or erroneous.
  4. Record the decision in code.

A negative quantity may be a legitimate refund.

“Unusual” and “wrong” are not synonyms.
5 · Safe assignment

Use loc to update rows selected by a condition

One indexing operation

mask = df["quantity"] >= 3

df.loc[
    mask,
    "order_size"
] = "bulk"

Chained assignment

df[mask]["order_size"] = "bulk"

This targets an intermediate object. It does not reliably update the parent; with pandas 3.0 Copy-on-Write, chained assignment never works.

6 · Boolean logic

Combine column conditions element by element

mask = (
    (df["quantity"] >= 3)
    & (df["date"] < "2026-06-01")
    & df["region"].isin(["East", "West"])
)

selected = df.loc[
    mask,
    ["transaction_id", "date", "quantity"],
]

Operators for Series

& means elementwise AND

| means elementwise OR

~ means elementwise NOT

Put parentheses around each comparison.

Python's scalar and/or are not substitutes for elementwise Series logic.
6 · Sorting and labels

Operations that depend on order

  • Previous value with shift
  • Forward fill with ffill
  • Rolling windows
  • Time-series returns

Sorting returns a new object unless you assign it back.

Row order is a property you must establish, not something to assume from the file.
6 · Derived columns

After cleaning, vectorized arithmetic produces an honest result

df["revenue"] = (

    * df["unit_price"]

    "unit_price",

What now works correctly

  • Numeric multiplication
  • Negative revenue for refunds
  • Missing price propagates to missing revenue
  • No explicit Python row loop
The expression did not change; the validity of the representation changed.
7 · pandas and NumPy

Use pandas for labeled structure and NumPy for homogeneous numerical arrays

pandas strengths

  • Column names and index alignment
  • Mixed column dtypes
  • Missing-data tools
  • Grouping, joining, reshaping

NumPy strengths

Do not convert the whole mixed table blindly

A DataFrame with strings, dates, and numbers may become an object array. That loses the performance and type clarity you wanted.

copy=False does not guarantee zero-copy conversion.

Choose the numeric block explicitly and check its shape and dtype.
7 · What conversion removes

A NumPy array carries positions—not DataFrame labels

Before conversion

After conversion

7 · Vectorized conditions

Use masks or np.where for array-wide conditional computation

Boolean mask

Conditional result
labels = np.where(
    revenue < 0,
    "refund",
    "sale",
)

df["transaction_type"] = labels
np.vectorize is mainly a convenience wrapper; it is not itself a promise of faster computation.
7 · Broadcasting

Broadcasting applies compatible smaller shapes across larger arrays

X = np.array([
    [10.0, 20.0, 30.0],
    [12.0, 18.0, 33.0],
    [ 8.0, 22.0, 27.0],
])

column_means = X.mean(axis=0)
centered = X - column_means

print(X.shape)             # (3, 3)
print(column_means.shape)  # (3,)

Compatibility rule

Compare dimensions from the right. Dimensions are compatible when they are equal or one of them is 1.

Broadcasting can be efficient, but very large temporary arrays can increase memory use.

The smaller array is conceptually repeated; NumPy usually performs the operation without materializing those repeats.

7 · Reshaping and time

Dates and pivots turn long observations into numerical matrices

df["month"] = (
    df["date"].dt.to_period("M")

monthly = (

Why wide matrices matter

Rows can represent dates; columns can represent customers, tickers, genes, or features.

The resulting numeric block can support correlation, returns, normalization, or matrix operations.

Always preserve the row and column labels.

Reshaping changes representation, not the underlying observations; verify that aggregation did not create or discard unintended values.
7 · Combining tables

A merge is a claim about keys and cardinality

sales = transactions.merge(
    customers,
    on="customer_id",
    how="left",
    validate="many_to_one",
    indicator=True,
)

print(sales["_merge"].value_counts())
print(len(transactions), "->", len(sales))

Declare the expectation

many_to_one says many transactions may share a customer, but each customer key appears once in the customer table.

indicator=True identifies unmatched rows.

A join can drop rows, create missing annotations, or multiply rows. None of those outcomes must remain silent.
7 · Reconciliation

Audit row counts and totals after every join and group

Two silent join failures

  • An inner join drops unmatched transactions.
  • Duplicate keys on both sides create a many-to-many multiplication.
before = transactions["revenue"].sum()

after = (
    sales.groupby(
        "region",
        dropna=False,
    )["revenue"]
    .sum()
    .sum()
)

assert np.isclose(before, after)
Reconciliation connects the summary back to the source and catches silent row loss or inflation.

The reproducibility contract

  1. Never hand-edit or overwrite source evidence.
  2. Record every assumption and repair in code.
  3. Rerun from the top to regenerate processed files.
  4. Reload exports and validate their contract.
If an output cannot be regenerated from raw evidence and code, it is not a trustworthy analytical artifact.
