How should a program represent data?

Data types, objects, structures, and storage

Choose representations that make the required operations clear and preserve the meaning of the data.

Mutability determines whether an object can change in place

Mutable objects

Immutable objects

3 · Copying

A copy creates a new outer container

original = [100, 200]
copied = original.copy()

copied.append(300)

print(original)  # [100, 200]
print(copied)    # [100, 200, 300]

copy() is shallow

It creates a new outer list or dictionary, but nested mutable objects may still be shared.

copy.deepcopy() recursively copies nested content when that behavior is truly needed.

Ask whether you need another name, another outer container, or a recursively copied structure.
3 · Copying · Predict

Does a shallow copy protect nested data?

original = {
    "tags": ["new", "email"]
}
copied = original.copy()

copied["tags"].append("vip")
print(original["tags"])
The dictionaries are different objects. What happens to the list stored inside them?
["new", "email", "vip"]. The outer dictionaries are separate, but both still refer to the same nested list.
Copy nested objects explicitly—or use copy.deepcopy() when supported nested contents should be recursively copied.

4 · Abstract data types

Queue: first in, first out

Request ARequest BRequest C

serve ← oldest request leaves from the left
